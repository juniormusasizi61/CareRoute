1.1  What CareRoute Does
CareRoute is a SaaS logistics dispatch platform built for remote field service managers. A dispatcher uploads
a list of client addresses and worker availability windows for the day. The backend geocodes every address,
applies a constraint-satisfaction route-optimisation algorithm (Google OR-Tools VRP solver), and returns an
optimised sequence of stops per worker. Routes are visualised on an interactive Mapbox map, and field
workers receive their schedule on a mobile-responsive web view that updates in real time as jobs are
completed, rescheduled, or cancelled.
The platform proves mastery of geospatial data (PostGIS), combinatorial optimisation, JWT-secured
multi-role APIs, real-time WebSocket state sync, and professional DevOps practice. These are skills at the
top of every senior engineering job description.

1.4  Data Flow Walkthrough
Every request in CareRoute follows this path from dispatcher input to worker map update:
1. Input
Dispatcher uploads CSV of addresses + worker schedules via React form
2. Geocoding
API calls Mapbox Geocoding API; stores lat/lng geometry in PostGIS
3. Matrix fetch
API requests a travel-time distance matrix (OSRM or Mapbox Directions)
4. Optimise
API enqueues a job; Python optimiser service pulls it, runs OR-Tools VRP, returns ordered stop
lists
5. Persist
Optimised routes written to PostgreSQL with PostGIS geometry columns
6. Broadcast
Socket.io emits route_ready to the dispatcher's room; map re-renders
